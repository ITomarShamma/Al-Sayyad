# معمارية متجر الصَّيَّاد

> وثيقة مرجعية للشركة: كيف مبني المشروع ولماذا. ترافق `database-schema.md`.
> آخر تحديث: 2026-07-04.

## الأساس التقني (ولماذا)

| القرار | السبب |
|---|---|
| **Django 5.2 + HTMX** | كود Python مقروء، لوحة تحكم جاهزة قوية، صفحات تُرسم بالسيرفر (أخف وأبسط)، وHTMX يعطي التفاعلية الحية بلا SPA معقّدة |
| **CSS يدوي + متغيّرات** | الهوية البصرية معرّفة مرة واحدة (`tokens.css`)؛ لا تبعية لأطر CSS |
| **أصول مستضافة محلياً** | الخط وhtmx داخل المشروع — لا CDN خارجي (أسرع وأضمن داخل سوريا) |
| **SQLite تطويراً / PostgreSQL إنتاجاً** | صفر إعداد محلياً؛ نفس الكود يشتغل على الاثنين |
| **سلة بالجلسة، بلا حسابات** | أول تجربة شراء أونلاين لمعظم الزبائن — أي حاجز تسجيل يقتل المبيعات |

## خريطة التطبيقات (apps)

```
apps/
├── core/      أساس مشترك: TimeStampedModel (created/updated لكل الجداول)
├── pages/     الرئيسية، عن الصياد، تواصل معنا، robots.txt، دليل المكوّنات
├── catalog/   التصنيفات (شجرة)، المنتجات، الصور+المصغّرات، البحث العربي، sitemap
├── cart/      السلة بالجلسة + عدّادات الهيدر الحية (HTMX OOB)
└── orders/    إتمام الطلب، الطلبات (لقطات أسعار)، التتبع، بوابات الدفع
```

**قاعدة التقسيم:** كل app مجال عمل مستقل يملك نماذجه وواجهاته وقوالبه واختباراته.
`pages` يجمّع للعرض فقط؛ المنطق الحسّاس يسكن بمجاله (مثلاً إنشاء الطلب في
`orders/services.py` وليس في أي view).

## مسار طلب نموذجي (مثال: «أضف للسلة»)

```
المتصفح ─ POST /cart/add/12/ (HTMX أو فورم عادي)
   └→ config/urls.py → apps/cart/urls.py → views.add
        ├─ يتحقق أن المنتج مفعّل (وإلا 404)
        ├─ Cart(request).add(product) ← يقصّ الكمية على المخزون ويكتب بالجلسة
        └─ الردّ حسب النوع:
             HTMX  → أجزاء OOB: عدّادا الهيدر + توست «أُضيف للسلة»
             عادي  → redirect لصفحة السلة (يعمل بلا JavaScript)
```

هذا «التحسين التدريجي» قاعدة عامة: **كل شيء يعمل بلا JS، وHTMX يحسّنه**.

## القواعد المعمارية الثابتة

1. **Tokens ثم مكوّنات ثم صفحات:** أي قيمة بصرية من `static/css/tokens.css`
   حصراً؛ أي قطعة UI تتكرر تصير مكوّناً في `templates/components/` (يُستدعى
   بـ `{% include ... only %}`) وتُعرض بحالاتها في `/styleguide/`.
2. **المفعّل فقط يظهر:** `is_active=False` = غير موجود للزبون (404) — تُفرض
   في queryset الـview، لا بشروط القوالب.
3. **لقطات لا مراجع:** سطر الطلب يخزّن اسم المنتج وسعره لحظة الشراء —
   الفواتير لا تتغيّر بأثر رجعي.
4. **الحذف محسوب:** PROTECT حيث للبيانات قيمة مستقلة (منتج↔تصنيف،
   منتج↔سطر طلب)، CASCADE حيث الابن بلا معنى وحده (صور المنتج، أسطر الطلب).
5. **منطق العمل في services:** تحويل السلة لطلب (قفل مخزون، خصم، تفريغ)
   دالة واحدة `@transaction.atomic` تُختبر لوحدها وتُستدعى من أي واجهة.
6. **عربي أولاً، إنجليزي جاهز:** كل النصوص عبر `{% translate %}`، خصائص CSS
   منطقية تقلب مع `dir`، والإنجليزية تتفعّل بإضافة ترجماتها (انظر أدناه).
7. **التطبيع قبل المقارنة:** البحث والهواتف يطبَّعان (أشكال الألف، ة/ه،
   الأرقام الهندية…) — ما يكتبه المستخدم بأي شكل يجب أن يُفهم.
8. **اختبار لكل موديول:** `python manage.py test apps` — كل ميزة دخلت مع
   اختباراتها، والأخطاء المصلَّحة تُثبَّت باختبار انحدار.

## الأدوار (من هو مَن)

| الدور | كيف يتحقق | ماذا يرى |
|---|---|---|
| **زائر Guest** | لا حساب إطلاقاً | يتصفح ويطلب ويتتبع برقم الطلب+الموبايل؛ طلبه `user=NULL` |
| **زبون** | حساب بالموبايل (`Profile.role=customer`) | «حسابي»: بياناته وطلباته، بيانات الطلب تتعبّى مسبقاً |
| **تاجر** | طلب انضمام + موافقة الإدارة (`MerchantProfile.is_approved`) | لوحة التحكم — **منتجاته فقط** (عزل في ProductAdmin) |
| **موظفو المتجر** | is_staff + مجموعة (مدير المتجر / موظف الطلبات / مدخل بيانات) | حسب صلاحيات مجموعته |
| **المدير العام** | superuser (عمر) | كل شيء |

لوحة التحكم نفسها بهوية الصَّيَّاد (`templates/admin/base_site.html` +
`static/css/admin-theme.css` — متغيّرات CSS الخاصة بـDjango admin).

## القادم (وما يلزمه)

- **شام كاش:** نقطة الوصل جاهزة في `apps/orders/payments.py`
  (`ENABLED_METHODS` + هيكل `ShamCashGateway`). ينتظر API والهوية البصرية.
- **الإنجليزية:** الكود جاهز بالكامل. المتطلب الوحيد: تنصيب GNU gettext على
  ويندوز (`choco install gettext` كمسؤول) ثم:
  `django-admin makemessages -l en` → ترجمة `locale/en/LC_MESSAGES/django.po`
  → `compilemessages` → إضافة `("en", "English")` في `LANGUAGES`.
- **النشر:** VPS + Gunicorn/Nginx + PostgreSQL (`config/settings/prod.py`
  جاهزة وتُقرأ من `.env`). بعض المزوّدين يحجبون IP سوري — يُختار المزوّد وفق ذلك.
