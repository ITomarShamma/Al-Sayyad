# نظام تصميم الصَّيَّاد (DESIGN)

> دليل الهوية البصرية وتجربة الاستخدام — الجيل الثاني: «رحلة الصَّيَّاد».
> من أين تأتي كل قيمة، وكيف تبني واجهة جديدة تشبه الموقع. ترافق
> `architecture.md` · العرض الحي لكل المكوّنات وحالاتها: `/styleguide/` ·
> آخر تحديث: 2026-07-15 (إعادة بناء الهوية من الصفر).

## القصة: رحلة الصَّيَّاد

الهوية الجديدة تروي قصة العلامة نفسها: **الصَّيَّاد يُبحر بالليل ويعود مع
الفجر حاملاً الصيد.** كل الألوان مشتقة من هذه الرحلة:

- **ليل البحر (كحلي عميق)** — الإطار الراسخ: الهيدر والفوتر والأسطح الداكنة.
- **البحر (أزرق)** — كل ما يُنقر ويُتفاعل معه.
- **ذهب الفجر (كهرماني)** — الجائزة: الفعل الأهم في الشاشة فقط.

## الفلسفة: خمس قواعد ذهبية

صُمّم المتجر لزبون قد يشتري أونلاين **لأول مرة بحياته**. كل قرار بصري
يخدم هذا:

1. **كل شاشة = مهمة واحدة.** صفحة إتمام الطلب لا تسوّق، وصفحة المنتج لا
   تشتّت. إن احتجت شرح الشاشة بجملتين فهي شاشتان.
2. **Tokens ثم مكوّنات ثم صفحات.** أي قيمة بصرية من `tokens.css` حصراً؛
   أي قطعة UI تتكرر تصير مكوّناً في `templates/components/` يُستدعى
   بـ`{% include ... only %}` ويُعرض بحالاته في `/styleguide/`.
3. **ما في أيقونة بدون كلمة.** 🛒 وحدها لغز لمن لا يعرف الأعراف — «🛒 السلة»
   واضحة للجميع.
4. **محايد الاتجاه.** خصائص CSS منطقية فقط (`inline-start` لا `left`) —
   الواجهة تنقلب تلقائياً بين العربية RTL والإنجليزية LTR.
5. **أظهر الثقة والحالة دائماً.** نقاط ثقة بالـhero، طرق الدفع بالفوتر،
   حالة الطلب خط زمني، كل فعل يعقبه تأكيد (توست/رسالة)، وكل خطأ يقال
   بصوت عالٍ وبالعربية.

## الألوان

**المصدر الوحيد:** `static/css/tokens.css`. لا يُكتب hex خارج هذا الملف.
كل الأزواج النصية مدقَّقة **WCAG AA** — لا تغيّر لوناً دون إعادة فحص التباين.

### ألوان البراند

| Token | القيمة | الدور |
|---|---|---|
| `--color-midnight` | `#0C2233` | ليل البحر: الهيدر والفوتر ورأس اللوحة |
| `--color-harbor` | `#14385C` | المرفأ: عمق التدرّجات، النص المالي، عناوين البطاقات |
| `--color-sea` | `#0F62B0` | **الأساسي**: أزرار، روابط، الحالة النشطة |
| `--color-sea-deep` | `#0B4C8C` | hover للأساسي |
| `--color-dawn` | `#F6A821` | **التمييز باقتصاد**: CTA الرئيسي، عدّاد السلة، النجوم، هالة «الحالي» |
| `--color-dawn-deep` | `#E08F00` | hover للتمييز |
| `--color-mist` | `#E9F1F9` | ضباب الصباح: خلفيات هادئة (صور المنتجات، الاختيار، hover) |

### المحايدة (باردة — من عائلة الأزرق)

| Token | القيمة | الدور |
|---|---|---|
| `--color-ink` | `#101C28` | النص الأساسي |
| `--color-slate` | `#51637A` | نص ثانوي/تلميحات |
| `--color-line` | `#DFE7EF` | حدود وفواصل |
| `--color-cloud` | `#F4F7FA` | خلفية الصفحة (رمادي سماوي، ليس أبيض ساطعاً) |

### الدلالية + خلفياتها (tints)

لكل دلالة ثلاثية: لون قوي (يصلح نصاً فوق الأبيض) + خلفية فاتحة + نص داكن
يُقرأ فوقها — `--color-success/error/warning/info` (`#15803D` / `#D93526` /
`#B45309` / `#1D6FC2`) مع `--tint-*` و`--on-tint-*`. تُستعمل معاً دائماً
(شارة، رسالة نظام، تنبيه فورم) — لا تُفرد الخلفية بلا نصّها المرافق.
ألوان الدفع: `--color-shamcash` `#128A64` و`--color-cod` `#7A5B2A`.

### أدوار لا أسماء

الواجهة تستعمل **الأدوار**: `--bg`, `--surface`, `--surface-dark`, `--text`,
`--text-muted`, `--border`, `--primary`, `--primary-strong`, `--strong`,
`--accent`, `--accent-strong`, `--on-primary`, `--on-accent`, `--tint`.
لو تغيّرت الهوية غداً يتغيّر تعريف الدور بسطر — لا مطاردة ألوان بالملفات.

### التدرّجات (Gradients)

معرّفة جاهزة في tokens — لا تُركَّب يدوياً بالملفات:
`--grad-brand` (ليل ← مرفأ ← بحر، للـhero) · `--grad-header` ·
`--grad-primary` (زر أساسي) · `--grad-accent` (زر CTA) ·
`--grad-mark` (علامة «ص») · `--grad-underline` (خط عناوين المقاطع).

### قرارات لونية مقصودة

- **الهيدر والفوتر Midnight:** الصفحة تُفتح وتُقفل بنفس الإطار الغامق —
  حضور راسخ يترك الأزرق البحري لما يُنقر. نفس رأس لوحة التحكم.
- **الذهبي بالقطّارة:** كثرة الذهبي تقتل قيمته؛ هو للفعل الأهم بالشاشة
  (CTA واحد، عدّاد السلة، هالة الخطوة الحالية).
- **سعر التخفيض أحمر** (`product-card__price--sale`) — يُقرأ فرصةً،
  والسعر القديم مشطوب بجانبه رمادياً.
- **الظلال كحلية لا سوداء:** كل الظلال مصبوغة بـMidnight بشفافية —
  عمق ينتمي للهوية بدل رمادي محايد باهت.

## الطباعة

الخط: **IBM Plex Sans Arabic** مستضاف محلياً (`fonts.css` — لا CDN).

| Token | الحجم | الاستخدام |
|---|---|---|
| `--fs-display` | 52px | عنوان الـhero على الشاشات الكبيرة فقط |
| `--fs-h1` | 36px | عنوان الصفحة (واحد لكل صفحة) |
| `--fs-h2` | 26px | عناوين المقاطع (بخط سفلي متدرّج) |
| `--fs-h3` | 18px | عناوين فرعية وبطاقات |
| `--fs-body` | 16px | النص الأساسي — **لا نص تحت 13px أبداً** |
| `--fs-caption` | 13px | تلميحات وتواريخ وشروح |

أوزان: 400 نص · 500 تسميات · 600 عناوين فرعية وأزرار · 700 عناوين (h1+h2)
وأسعار. الأسطر: `--lh-base` 1.75 للنص العربي و`--lh-tight` 1.25 للعناوين.
**ممنوع `letter-spacing` على العربية** — يكسر اتصال الحروف.
**الأرقام المالية تُنسَّق بالسيرفر** (`f"{...:,.0f}"`) — القالب تحت اللغة
العربية يطبع 4.0 كـ«٤٫٠» (درس M24).

## المسافات والأشكال والعمق

- **المسافات:** سلّم أساسه 4px — `--space-1..10` (4/8/12/16/24/32/40/48/64).
  لا قيم حرة؛ القرب البصري = علاقة منطقية.
- **الزوايا:** `--radius-sm` 8 (مدخلات صغيرة) · `--radius-md` 14 (بطاقات
  وحقول) · `--radius-lg` 20 (بطاقات المنتجات والدوك) · `--radius-xl` 28
  (hero ومعرض الصور) · `--radius-pill` (أزرار وشارات).
- **الظلال — ثلاث طبقات عمق، كل ظل طبقتان** (قريبة حادة + بعيدة ناعمة)
  مصبوغة كحلياً: `--shadow-sm` للبطاقات الساكنة · `--shadow-md` للمرفوعة
  hover · `--shadow-lg` لأعلى طبقة (اقتراحات البحث، التوست، الدوك).
  وتوهّجان ملوّنان للأزرار: `--glow-primary` و`--glow-accent`.
- **الحركة:** `--transition-fast` 0.15s (ألوان/حدود) و`--transition-base`
  0.25s بمنحنى `cubic-bezier(0.2,0.7,0.3,1)` (رفعة/تكبير).
  `prefers-reduced-motion` يوقف كل حركة — دائماً.

## البنية (Layout)

- الحاوية `--container-max` 1200px، النصوص والفورمات `--content-narrow` 720px.
- نقطة تحوّل واحدة: **768px**. تحتها: بحث بسطر كامل + **دوك سفلي عائم**
  (شريط مموّه بزوايا كبيرة يطفو فوق الحافة، العنصر النشط داخل حبّة Mist،
  أهداف نقر `--tap-min` 48px). فوقها: تنقّل علوي بالهيدر ولا دوك.
- الهيدر لاصق (sticky) بتدرّج Midnight؛ حقل البحث «زجاجي» يشتعل بحدّ
  ذهبي عند التركيز؛ التبويب النشط حبّة بصبغة الفجر.
- الفوتر Midnight غامق: ثلاثة أعمدة على الشاشات الكبيرة (الهوية / روابط /
  طرق الدفع) وعمود واحد ممركز على الموبايل، وتحته سطر الحقوق ومبدّل اللغة
  (شريحة زجاجية داخل الفوتر).
- الـhero: تدرّج `--grad-brand` + توهّج فجر بزاوية + تموّجات ماء (حلقات
  CSS) — وفيه: شريحة تمهيدية زجاجية، عنوان display، سطران CTA (ذهبي +
  شبح) ونقاط ثقة بعلامة ✓ ذهبية.
- عناوين المقاطع: خط سفلي قصير متدرّج (فجر ← بحر) بدل العلامة الجانبية.
- شبكة المنتجات: `auto-fill minmax(150px)` موبايل / `minmax(210px)` فوق 768.

| اقتراحات البحث | `.search-suggest` | تحت الحقل مباشرة، `--shadow-lg` |

تُستدعى بـ`only` ← أضفه بحالاته إلى `/styleguide/` ← استعمله.

## أنماط التفاعل

- **الاستجابة قبل النقر:** كل قابل للنقر له hover ملموس (رفعة/لون/توهّج).
- **الضغطة:** الأزرار ترتفع 1px عند hover وتنزل 1px عند `:active`.
- **التركيز:** حلقة `:focus-visible` موحّدة (Sea) بكل الموقع — تظهر
  للكيبورد فقط لا للفأرة. داخل الأسطح الداكنة (هيدر/hero): الحلقة ذهبية
  (`--focus-ring-accent`).
- **التأكيد:** إضافة للسلة = توست + تحديث عدّادين حيّين؛ حفظ = رسالة نجاح.
- **التحسين التدريجي:** كل شيء يعمل بلا JavaScript؛ HTMX يحسّن فقط.

## RTL والترجمة

- عربي أولاً؛ `dir` يتبدّل مع اللغة والـCSS المنطقي يقلب التخطيط وحده.
- كل نص جديد عبر `{% translate %}` ثم سير العمل:
  `makemessages` ← `scripts/translate_en.py` (يفشل معدِّداً الناقص) ←
  `compilemessages` (التفصيل في `development-log.md`).
- الأرقام الهندية (٠٩…) تُطبَّع أينما أدخلها المستخدم.

## الوصولية (Accessibility)

- أهداف نقر ≥ 48px (`--tap-min`)، نص ≥ 13px.
- كلمة مع كل أيقونة؛ `aria-label` للمناطق؛ `alt` لكل صورة منتج.
- `:focus-visible` واضح، `::selection` بهوية العلامة.
- `prefers-reduced-motion` يجمّد الحركة كلها.
- التباين: كل أزواج النص/الخلفية في tokens مدقَّقة AA (نص عادي ≥ 4.5)،
  بما فيها الأبيض فوق Sea وInk فوق Dawn والثلاثيات الدلالية.

## لوحة التحكم

نفس الهوية عبر متغيّرات Django admin نفسها — `static/css/admin-theme.css` +
`templates/admin/base_site.html`: رأس Midnight، أزرار Sea، أدوات ذهبية
(Dawn)، الوضع الفاتح مثبَّت، وخط المتجر نفسه. لوحة المبيعات
(`/admin/dashboard/`) تتبع نفس النظام (بطاقات، مخطط أعمدة CSS بأزرق
البحر، أشرطة ذهبية، ألوان الحالات الدلالية).
