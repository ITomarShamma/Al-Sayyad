# نظام تصميم الصَّيَّاد (DESIGN)

> دليل الهوية البصرية وتجربة الاستخدام — هوية **«شفق» (Aurora)**: ثيمان
> كاملان (فاتح + داكن)، لوحة شفقية، وحركة راقية بالتحسين التدريجي.
> من أين تأتي كل قيمة، وكيف تبني واجهة جديدة تشبه الموقع. ترافق
> `architecture.md` · العرض الحي لكل المكوّنات وحالاتها: `/styleguide/` ·
> آخر تحديث: 2026-07-15 (إعادة بناء الهوية من الصفر — تجربة Opus).

## القصة: شفق فوق البحر

الهوية تلتقط لحظة **الشفق**: السماء تتحوّل، والبحر يلمع تحتها. ثلاثة ألوان
تصنع هذا «الشفق» وتتحرّك ببطء في خلفية الـhero:

- **فيروزي البحر (Teal)** — اللون الأساسي: كل ما يُنقر ويُتفاعل معه.
- **مرجاني دافئ (Coral)** — الفعل: الـCTA الأهم في الشاشة فقط.
- **بنفسجي السماء (Violet)** — التوهّج والوسم: يظهر في التدرّجات والعلامة.

## الفلسفة: خمس قواعد ذهبية

صُمّم المتجر لزبون قد يشتري أونلاين **لأول مرة بحياته**. كل قرار بصري
يخدم هذا:

1. **كل شاشة = مهمة واحدة.** صفحة إتمام الطلب لا تسوّق، وصفحة المنتج لا
   تشتّت. إن احتجت شرح الشاشة بجملتين فهي شاشتان.
2. **Tokens ثم مكوّنات ثم صفحات.** أي قيمة بصرية من `tokens.css` حصراً؛
   أي قطعة UI تتكرر تصير مكوّناً في `templates/components/` يُستدعى
   بـ`{% include ... only %}` ويُعرض بحالاته في `/styleguide/`.
3. **ما في أيقونة بدون كلمة.** 🛒 وحدها لغز لمن لا يعرف الأعراف.
4. **محايد الاتجاه.** خصائص CSS منطقية فقط (`inline-start` لا `left`).
5. **أظهر الثقة والحالة دائماً.** نقاط ثقة بالـhero، طرق الدفع بالفوتر،
   خط زمني للطلب، تأكيد بعد كل فعل، وكل خطأ يقال بصوت عالٍ وبالعربية.

## الثيمان: فاتح وداكن (جديد)

النظام كلّه مبنيّ على **الأدوار** (`--bg`, `--surface`, `--text`, `--primary`…).
تُعرَّف مرّة للثيم الفاتح في `:root`، وتُعاد قيمها فقط للثيم الداكن — فكل
مكوّن ينقلب تلقائياً بلا سطر إضافي.

- **الحسم:** يتبع تفضيل النظام (`prefers-color-scheme`) افتراضياً، ويتجاوزه
  اختيار المستخدم عبر زر التبديل (يُخزَّن في `localStorage`).
- **بلا وميض:** سكربت مضمَّن صغير في `<head>` يضبط `data-theme` قبل أول رسم.
- **الهيدر والفوتر والدوك «زجاجية»** (`--glass-bg` + blur) تتكيّف مع الثيم —
  لا لون مفروض، فكلا الوضعين مقصود لا افتراضي.
- **تدقيق التباين:** كل أزواج النص/الخلفية مدقَّقة **WCAG AA في الثيمين**
  بسكربت فحص (النص العادي ≥ 4.5، والأساسي ≥ 7).

### الدلالية + خلفياتها (tints)

لكل دلالة ثلاثية: لون قوي + خلفية فاتحة + نص يُقرأ فوقها —
`--color-success/error/warning/info` مع `--tint-*` و`--on-tint-*` (لكلٍّ قيم
مستقلة في الثيمين). تُستعمل معاً دائماً. ألوان الدفع: `--color-shamcash`
و`--color-cod`.

### التدرّجات (جاهزة كـtokens)

`--grad-primary` / `--grad-accent` (أزرار) · `--grad-mark` (العلامة، فيروزي←بنفسجي)
· `--grad-underline` (خط عناوين المقاطع). الأزرار تضع لوناً صلباً قبل التدرّج
كسند للمتصفّحات القديمة (بلا `color-mix`).

## الطباعة

الخط: **IBM Plex Sans Arabic** مستضاف محلياً (`fonts.css` — لا CDN).

| Token | الحجم | الاستخدام |
|---|---|---|
| `--fs-display` | 56px | عنوان الـhero (شاشات كبيرة) |
| `--fs-h1` | 36px | عنوان الصفحة |
| `--fs-h2` | 26px | عناوين المقاطع (بخط سفلي متدرّج) |
| `--fs-h3` | 18px | عناوين فرعية وبطاقات |
| `--fs-body` | 16px | النص الأساسي — **لا نص تحت 13px** |
| `--fs-caption` | 13px | تلميحات وتواريخ |

عناوين بـ`letter-spacing: -0.01em` وأوزان 700 لحضور أوضح. **ممنوع
`letter-spacing` موجب على العربية** — يكسر اتصال الحروف. الأرقام المالية
تُنسَّق بالسيرفر (درس M24).

## الحركة (Motion) — تحسين تدريجي بالكامل

كل حركة **تُلغى تماماً** عند `prefers-reduced-motion`، والموقع يعمل بلا
JavaScript (HTMX/JS يحسّنان فقط). رموز الحركة في tokens: `--dur-1..4`
ومنحنيان `--ease-out` و`--ease-spring`.

- **شفق متحرّك:** ثلاث هالات لونية تنجرف ببطء خلف الـhero (`aurora-drift`).
- **كشف عند التمرير (reveal):** العناصر بكلاس `.reveal` تنزلق وتظهر عند
  دخولها الشاشة (`IntersectionObserver` + stagger). **بلا JS تبقى ظاهرة**
  (تُخفى مبدئياً فقط عند وجود `.js`).
- **ميلان البطاقات (tilt):** بطاقات المنتج تميل قليلاً مع المؤشّر
  (`[data-tilt]`) — على أجهزة المؤشّر الدقيق فقط.
- **عدّادات:** أرقام الـhero تعدّ للأعلى عند ظهورها (`[data-count]`).
- **لمسات:** لمعة تعبر الأزرار عند hover، هالة نابضة للخطوة الحالية في خط
  الزمن، شمر (shimmer) على صور المنتجات الفارغة، والعلامة «تتنفّس» بلطف.
- **الهيدر** يرقّ ويكتسب ظلاً عند التمرير (`is-scrolled`).

طبقة السلوك في `static/js/app.js` (صغيرة، بلا تبعيات): ثيم + reveal + tilt
+ عدّادات + رأس متجاوب.

## المسافات والأشكال والعمق

- **المسافات:** سلّم أساسه 4px — `--space-1..12`.
- **الزوايا:** `sm 8` (مدخلات) · `md 14` (حقول) · `lg 22` (بطاقات/دوك) ·
  `xl 32` (hero/معرض) · `pill`.
- **الظلال — ثلاث طبقات، لكل ظل طبقتان**، وتُعرَّف مستقلّةً لكل ثيم (أغمق
  وأعمق في الداكن). وتوهّجان ملوّنان للأزرار: `--glow-primary/-accent`.

## البنية (Layout)

- الحاوية `--container-max` 1200px، النصوص `--content-narrow` 720px.
- نقطة تحوّل واحدة **768px**. تحتها: بحث بسطر كامل + **دوك سفلي عائم زجاجي**
  (العنصر النشط داخل حبّة، أهداف نقر 48px). فوقها: تنقّل علوي ولا دوك.
- **الهيدر** زجاجي لاصق، وفيه زر تبديل الثيم؛ حقل بحث يتوهّج عند التركيز.
- **الـHero** قاعدة داكنة ثابتة (`--hero-base`) يطفو فوقها الشفق المتحرّك؛
  النص أبيض في الثيمين — لوحة درامية موحّدة. فيه: شريحة تمهيدية، عنوان
  display بكلمة مضيئة، CTA ثنائي، نقاط ثقة، وشريط عدّادات.
- **الرئيسية** تستعمل شبكة **bento**: بطاقة «موثوق» كبيرة (2×2) بخلفية شفقية،
  وبقيّة البطاقات حولها.
- **الفوتر** سطح متكيّف بخطّ شفقي علوي رفيع.

## المكوّنات (components.css + templates/components/)

| المكوّن | الكلاس | ملاحظات |
|---|---|---|
| زر | `.btn` + `--primary/--accent/--secondary/--ghost/--block` | لمعة تعبره عند hover، رفعة 2px، توهّج ملوّن. `--ghost` للأسطح الداكنة |
| شارة | `.badge` + دلالات ودفع | ثلاثية اللون، تعمل بالثيمين |
| حقل | `.field` | حد 1.5px، تركيز بحلقة Teal |
| بطاقة | `.card` (+`--link`/`--feature`) | `--feature` بخلفية شفقية لبطاقة bento البارزة |
| بطاقة منتج | `.product-card` | كشف + ميلان + تكبير صورة + شمر placeholder + `sale-flag` مرجانية |
| شريحة | `.chip` | ترتفع 2px عند hover |
| توست | `.toast` | ينزلق بارتداد، رابطه مرجاني، `aria-live` |
| خيار دفع | `.pay-option` | المختار بحدّ Teal مضاعف + خلفية غائرة |
| خط زمن | `.timeline` | الحالي بهالة **نابضة**، الملوَّن Teal |
| رسائل | `.messages__item--*` | حدّ جانبي بلون الدلالة + انزلاق دخول |

← أضفه لـ`/styleguide/` ← استعمله. تحقّق أنه سليم في الثيمين.

## RTL والترجمة

- عربي أولاً؛ `dir` يتبدّل مع اللغة والـCSS المنطقي يقلب التخطيط وحده.
- كل نص جديد عبر `{% translate %}` ثم: `makemessages` ←
  `scripts/translate_en.py` ← `compilemessages`.

## الوصولية (Accessibility)

- أهداف نقر ≥ 48px، نص ≥ 13px، كلمة مع كل أيقونة، `aria-label`/`alt`.
- زر الثيم `aria-pressed`؛ `:focus-visible` واضح (ذهبي داخل الأسطح الداكنة).
- **`prefers-reduced-motion` يجمّد كل شيء** ويُظهر عناصر الكشف فوراً.
- **التباين مدقَّق AA في الثيمين** — لا يتغيّر لون دون إعادة الفحص.

## لوحة التحكم

نفس الهوية عبر متغيّرات Django admin — `static/css/admin-theme.css`: رأس داكن،
أزرار Teal، أدوات مرجانية، علامة «ص» بتدرّج الشفق، مخطط مبيعات بأعمدة Teal
وأشرطة مرجانية. الوضع الفاتح مثبَّت للوحة (المتجر وحده ثنائي الثيم).
